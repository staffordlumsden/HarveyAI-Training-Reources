Teaching Materials with HarveyAI

Use this walkthrough to design draft teaching and assessment-support materials for educator review. The page keeps the work bounded, source-aware, and tied to verification before anything is used with students.

Source PDF Home

Start with the teaching problem, not the tool.

Harvey can be used to produce draft teaching or assessment-support materials for educator review. The output is not ready for direct release until it has been checked for legal accuracy, pedagogical fit, source grounding, and student risk.

Purpose

Support teaching preparation, seminar design, revision activities, case explanation, or assessment-support drafting without treating unverified output as authority.

Safe pattern

Define purpose, provide context and source limits, generate a structured draft, verify, revise, and document the educator’s decisions.

Boundary

Do not use Harvey to bypass legal judgement, educator review, source checking, copyright obligations, or privacy requirements.

Working principle

Use Harvey to draft and organise material for review. The educator determines what is accurate, appropriate for the cohort, and suitable for the intended learning activity.

Select a bounded teaching-material task.

Choose one task that can be reviewed against known sources and a clear teaching purpose. The page will generate a draft prompt that you can edit before testing in Harvey.

Planning sequence

  1. Decide what the material is meant to support.
  2. Select the smallest useful teaching output.
  3. Name the unit, cohort, jurisdiction, topic, class format, and level of detail.

What to avoid

Avoid broad requests such as “make teaching materials”. Ask for a defined output: an IRAC summary, seminar sequence, issue-heading scaffold, short explainer, teaching note, or revision activity.

Selection rule

Select the task that is easiest to verify and most clearly connected to student learning. A narrower prompt usually gives the educator more control.

Prepare the source material and set the limits.

Harvey output is easier to review when it is tied to supplied material. Use a bounded corpus and state what the output must not do.

Source materials

  • Case, extract, or judgment.
  • Statute provision or problem scenario.
  • Existing lecture notes or reading list.
  • Relevant Vault corpus, where appropriate.

Limits to state

  • Do not invent authorities.
  • Do not provide legal advice.
  • Do not exceed the jurisdiction.
  • Do not produce a complete model answer unless explicitly required.
  • Flag uncertainty and verification points.
Upload boundary

Before uploading or pasting source materials, check that the material does not conflict with copyright obligations and does not contain personally identifiable information. If working from Vault, use only the relevant corpus.

Draft prompt for preparation

Generate a structured draft, then verify it.

Ask for a defined output format, then check the result before it becomes a teaching artefact. Verification is the point at which educator judgement remains visible.

Structured output

Ask for headings, a table, sequence of questions, short teaching note, IRAC structure, issue map, discussion plan, or annotated outline.

Legal accuracy

Verify citations, propositions, quotations, doctrinal characterisations, and jurisdictional claims against primary sources or approved legal databases.

Pedagogical suitability

Check whether the draft supports the intended learning outcome, matches the cohort level, and avoids giving students a complete answer where that would undermine learning.

Verification focus

Record at least one verified strength, one correction, and one unresolved issue. These notes make the activity useful for teaching rather than merely plausible.

Revise the draft into a teaching activity and document the process.

Edit the output before use. Where possible, convert the material into an activity that requires students to do legal reasoning, not simply consume an answer.

Revise for student use

Edit tone, remove unnecessary complexity, add prompts for active learning, and make the legal reasoning task visible. A handout may be less useful than an activity.

Document verification

Keep a short note recording what Harvey was used for, what sources were checked, what changed, and what remains educator-authored.

Copy a prompt template and adapt it before testing.

These templates are starting points. Before copying to Harvey, name the unit, cohort, jurisdiction, topic, source materials, and intended teaching use.

Final prompt from selected task

        

        


        

          
IRAC-style case summary for teaching

          
Using only the supplied case materials, draft a concise IRAC-style teaching summary for [UNIT] students studying [TOPIC] in [JURISDICTION].

Audience: [FIRST-YEAR LLB / LATER-YEAR LLB / JD / LLM].
Purpose: support class preparation and discussion, not replace student reading.

Provide:
1. Material facts
2. Procedural posture
3. Legal issue
4. Rule or governing principle
5. Application / reasoning
6. Holding
7. One teaching point
8. Two questions for class discussion
9. Matters requiring verification

Do not invent authorities. Do not add propositions not supported by the source materials. Flag uncertainty.

        


        

          
Seminar discussion questions

          
Draft a sequenced set of seminar discussion questions for [UNIT] on [TOPIC].

Audience: [STUDENT COHORT].
Jurisdiction: [JURISDICTION].
Source materials: use only [SPECIFY MATERIALS].

Provide:
1. One opening recall question
2. Two application questions
3. One comparison or evaluation question
4. One likely misconception
5. One closing consolidation activity

Do not provide model answers. Identify which source or legal concept each question is designed to activate.

        


        

          
Updating lecture notes with recent case law

          
Compare the supplied lecture notes with the supplied recent case materials.

Task:
1. Identify where the lecture notes may need updating.
2. Suggest revised wording using tracked-change style markers: [DELETE: ...] and [ADD: ...].
3. Provide the source case and paragraph reference for each proposed change.
4. Flag any suggested change that requires educator verification.

Do not change the pedagogical structure unless necessary. Do not invent authorities or assume developments beyond the supplied materials.

        


        

          

          

            Completion point
            
The workflow is complete when the educator can explain the teaching purpose, the source boundary, the verification process, and how the final material preserves student legal reasoning.
