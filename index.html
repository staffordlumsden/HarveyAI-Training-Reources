Harvey Familiarisation Hub

A front page for five educator-facing Harvey resources. Use it to orient staff, link the pages together, and keep the emphasis on bounded tasks, verification, and educator judgement.

Introduce the collection as a disciplined workflow, not a set of demonstrations.

This landing page positions the five resources as a coherent pathway for teaching staff: manage reusable prompts, describe repeatable workflows, set up bounded Vault workspaces, prepare teaching materials, and practise assessment-support workflows with educator verification.

Purpose

Use this page as the entry point for a Harvey familiarisation site published through GitHub Pages, Netlify, SharePoint, or Canvas. It introduces the resources without treating Harvey output as authority.

Educator action

Use the Prompt Library and Build studios to frame reusable work, use the Vault walkthrough for bounded source work, then move into the Teaching Materials and Assessment Support walkthroughs for task-specific practice.

Boundary

Each page supports educator review, Vault source work, assessment design, structured feedback, workflow description, or prompt governance. It should not be used to create unchecked legal advice or unsourced doctrinal claims.

Working principle

Harvey can be used to organise, extract, compare, draft, and structure material for review. The educator remains responsible for task design, verification, and any student-facing use.

Verification over spectacle

The pages should help educators preserve legal and pedagogical judgement. Animations, tabs, and interface effects are only signposts.

Section 2 of 3

Give educators a sensible route through the resources.

The pathway keeps the work human-led: first define the teaching purpose, then use Harvey to support structured work, then verify outputs, and only then consider repeatable workflows, Vault source work, or shared prompt assets.

1

Govern reusable prompts.

Use Harvey Prompt Library Management Studio to localise, restrict, record, or retire prompts before they become shared resources.

2

Automate cautiously.

Use Harvey Build Workflow Studio only for repeatable tasks with clear inputs, low-risk test material, verification steps, and a defined human review point.

3

Set up source work.

Use Harvey Vault Set Up and Use to create a bounded corpus, build a Review Table, and verify claims before teaching use.

4

Prepare teaching materials.

Use Teaching Materials with HarveyAI to turn a bounded teaching problem into structured drafts, activities, and prompt templates for educator review.

5

Practise assessment support.

Use Harvey Assessment Support Workflow to model how evidence is mapped to criteria, how patterns are identified, and what must be checked.

Teaching move

Ask educators to record one verified strength, one correction, and one unresolved issue after each test. Those notes make the activity useful for learning design rather than simply presentable as a tool demonstration.

Before testing

  • Name the unit, cohort, jurisdiction, and intended teaching use.
  • Specify the input material and output format.
  • Define what should not be inferred.

During testing

  • Use low-risk or approved material.
  • Keep the task bounded and repeatable.
  • Require uncertainty to be flagged.

After testing

  • Check the output against primary or approved sources.
  • Record corrections and unresolved issues.
  • Decide whether the workflow should be reused, revised, or abandoned.
