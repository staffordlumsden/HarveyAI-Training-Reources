Harvey Assessment Support Workflow

Use this walkthrough to plan a disciplined assessment-support workflow: assessment design, feedback drafting, Review Table analysis, verification, and educator-led decision-making.

Source PDF

Start with the assessment purpose, not the tool.

Harvey can be used to support assessment design, feedback drafting, rubric mapping, and large-scale pattern analysis. The educator determines final standards, legal accuracy, feedback quality, and assessment decisions.

Purpose

This resource helps educators plan a safe assessment-support workflow that remains bounded, reviewable, and aligned with the intended learning purpose.

Boundary

Do not use Harvey to make final academic judgements, assign final marks, or process identifiable student data outside approved guardrails.

Expected user action

Move through the tabs in order. Define the assessment task, set data boundaries, map evidence to criteria, analyse patterns, and verify before use.

Working principle

Use Harvey to draft, organise, extract, compare, and map material for review. Do not treat unverified output as authority or as an assessment decision.

Draft assessment criteria and standards for educator review.

Use this process when designing or revising assessment instructions, criteria, standards, rubrics, sample outlines, or moderation materials.

Educator action

  1. Define the assessment-design problem.
  2. Gather task instructions, ULOs, criteria, grade descriptors, weighting, and policy constraints.
  3. State the cohort, level, jurisdiction, and intended student performance.
  4. Ask Harvey for a draft only.

Quality standard

The draft criteria should be assessable, distinct, aligned with the task, readable by students, and usable by more than one marker. Remove criteria that are vague, overlapping, or impossible to judge from the submission.

Verification focus

Check alignment against the assessment task, unit learning outcomes, course outcomes, AQF level, and local assessment rules before using the material.

Use feedback support to map evidence to criteria.

This process supports feedback drafting, rubric mapping, moderation, feed-forward advice, and cohort-level feedback. It is not a process for outsourcing marking decisions.

Prepare

Confirm permitted use, remove PII, and supply the assessment task, rubric, marking criteria, grade descriptors, and feedback conventions.

Map

Ask Harvey to identify evidence in the anonymised submission that appears relevant to each criterion. Require uncertainty flags.

Moderate

Compare each suggested comment with the original student work. Delete or revise comments that overstate, infer, or misclassify performance.

Data boundary

Remove names, student IDs, file metadata, comments, coversheets, email addresses, and other identifying information before upload.

Prompt: map anonymised submission to criteria
Using only the supplied anonymised student submission and supplied rubric, map evidence from the submission to the assessment criteria.

Output table:
1. Criterion
2. Evidence in the submission
3. Apparent strength
4. Apparent weakness or omission
5. Draft feedback comment for educator review
6. Human moderation note

Constraints:
Do not assign a final mark.
Do not determine a grade.
Do not infer beyond the supplied text.
Do not identify the student.
Flag any judgement that requires academic review.

Build a Review Table around assessment judgement.

Use Review Tables when working with multiple submissions or documents and you need structured extraction, pattern identification, moderation support, or cohort-level analysis.

Educator action

  1. Define the analysis purpose.
  2. Prepare a de-identified upload set.
  3. Upload the task, rubric, and anonymised submissions.
  4. Create custom extraction columns.
  5. Add manual verification and moderation columns.

What to avoid

Do not settle for a generic “summary” column. Build columns around the assessment task: issues identified, authorities cited, application quality, common errors, standout strengths, verification status, and moderator notes.

Column Purpose Example extraction instruction
Submission pseudonymTrack files without identifying students.Record the pseudonymous file label only. Do not extract names or student IDs.
Issues identifiedAnalyse issue spotting.List the legal issues identified in the submission. Use the student’s wording where possible.
Missing issuesIdentify common omissions.Identify legally relevant issues from the task that appear absent or underdeveloped.
Authorities citedReview research and authority use.List cases, statutes, regulations, and secondary sources cited.
Application qualityReview legal reasoning.Assess whether the submission applies law to facts with specificity. Do not assign a mark.
Verification statusPreserve human review.Manual column: verified / needs checking / incorrect / unsupported / not applicable.

Ask over the populated table for patterns, not decisions.

Once the Review Table is populated and checked, query the structured table for patterns, comparisons, and synthesis. The output should support moderation, feedback, and assessment redesign.

Ask

Use only the populated table. Ask for recurring strengths, recurring weaknesses, authority-use patterns, common omissions, and criteria that may need clarification.

Verify

Click through supporting citations and check whether each pattern is grounded in the original submissions or rubric.

Redesign

Use verified patterns to improve teaching, task instructions, criteria clarity, scaffolding, and feed-forward activities.

Prompt: ask over a populated Review Table
Using only the populated Review Table, identify the five most common patterns across the submissions.

For each pattern, provide:
1. Pattern name
2. Evidence from the relevant table columns
3. Why it matters for the assessment criteria
4. Draft cohort-level feedback
5. Suggested teaching or feed-forward response
6. Any limitation or uncertainty requiring educator verification

Do not refer to individual students.
Do not assign marks.
Do not make claims that are not supported by the table.

Completion point

The table is useful only when the educator can explain what was extracted, what was checked, what was corrected, and how the result will support student learning or moderation.

Confirm the workflow is ready for use.

Use this checklist before uploading, extracting, mapping, or releasing any Harvey-supported assessment or feedback output.

Bottom line

Design the assessment carefully, use Harvey to draft and structure support materials, use feedback workflows to map evidence to criteria, use Review Tables to identify cohort patterns, verify everything, and keep final academic decisions with educators.
