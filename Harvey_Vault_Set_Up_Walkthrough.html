Harvey Vault Set Up and Use

A Canvas-safe walkthrough for creating a bounded Vault corpus, building a Review Table, checking citations, and turning verified outputs into teaching materials.

Home Source PDF

Treat Vault as a controlled document workspace.

Vault is useful when an educator has a defined corpus and a defined teaching task. It should not be used as a shortcut around legal judgement, source checking, or educator responsibility.

Purpose

Create a bounded, searchable workspace for extracting, comparing, and verifying information from defined documents.

Human role

The educator determines the task, checks the sources, corrects the output, and decides whether anything is suitable for student-facing use.

Boundary

Do not treat Vault output as authority until it has been checked against the source documents and, where relevant, an external legal database.

Use cases

Use Vault for case comparison, assessment moderation, lecture-note updating, issue spotting, authority checking, or cohort-level analysis of anonymised submissions.

Working principle

Use Harvey to organise, extract, compare, or draft material for review. Do not use it to replace source verification, privacy checks, or legal method.

Section 1 of 6

Define the purpose and curate the corpus before upload.

The quality of a Vault task depends on what is included, what is excluded, and what the educator asks the workspace to support.

1. Define the purpose

Decide whether the Vault is for case comparison, moderation, lecture-note updating, issue spotting, authority checking, or cohort-level analysis.

2. Check the data boundary

For student work, remove names, student IDs, file metadata, comments, and any other personally identifiable information before upload.

3. Curate the corpus

Upload only the material needed for the task, such as two judgments, one statute, an approved source extract, or anonymised submissions.

Selection rule

A good Vault begins with a narrow use case and a safe upload set. Avoid uploading everything and hoping the workspace will impose structure later.

Create a named Vault and confirm the contents before analysis.

Begin with a workspace name that records the subject and purpose. Then upload only the selected files, allow processing, and run a source-identification check.

Section 3 of 6

Build the Review Table around legal components, not generic summary.

Use Review Tables where the task involves multiple files or repeated extraction. Columns should map to the legal and teaching work the educator intends to do.

Column design

Avoid generic columns like summary. Prefer issue, authority cited, statutory provision, reasoning, application, error, omission, evidence of synthesis, source paragraph, or teaching use.

Custom extraction

Write a precise instruction for each column. Ask for source references and require uncertainty to be flagged where the document does not support the cell.

Citation checking

Click through citations supporting each table cell. Verify extracted information against original documents and edit cells where needed.

Manual notes

Add educator comments, moderation notes, verification status, teaching use, or risk flags as manual columns.

Example extraction instruction 
Identify each authority cited in the document. Include the citation as written, the legal proposition for which it is used, and the paragraph or page reference if available.

Section 4 of 6

Ask across the populated table, then verify every claim.

Once the Review Table is populated, ask comparative or synthetic questions. Keep the prompt anchored to the table and require uncertainty to be visible.

Review Table extraction prompt 
For each document in this Vault, extract the case name or document title, court or source, legal issue, relevant statutory provisions, authorities cited, outcome, key reasoning, and source paragraph references. Keep each cell concise. Do not infer beyond the source text. Flag uncertainty where the document does not provide enough information.
Ask-across-table prompt 
Using only the populated table, identify the three most common issue-spotting omissions and give one anonymised example of each. Flag any uncertainty.
Verification prompt 
For each claim in the draft output, identify the supporting source document and paragraph, page, or citation. Flag any claim that is unsupported, incomplete, ambiguous, or requires checking against an external legal database.
Verification focus

Record at least one verified strength, one correction, and one unresolved issue. These notes make the activity useful for teaching rather than simply presentable as a demonstration.

Section 5 of 6

Translate verified Vault work into learning design.

Use verified outputs to draft seminar questions, worked examples, feedback themes, common-error notes, or revision activities. Avoid giving a complete model answer unless that is the intended teaching design.

Convert output

Turn checked table outputs into teaching artefacts that support comprehension, comparison, evaluation, or feedback.

Document workflow

Record what was uploaded, what Harvey was used for, what was checked, what was corrected, and what was not relied upon.

Share carefully

Share a Vault or table only with the correct group and only after confirming data permissions, copyright or licence conditions, and teaching relevance.

Teaching-use prompt 
Using only the verified Review Table, draft five seminar discussion questions for later-year Australian law students. The questions should move from comprehension to comparison and evaluation. Do not provide model answers. Identify which table columns support each question.
Audit note

A short audit note supports accountable use and keeps disclosure, verification, and human-led judgement visible.
